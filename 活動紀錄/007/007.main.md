# 命名 in CSS
- 原則

## CSS 命名常見較普遍的原則
- 可預測性：看到命名可預測大致樣式。
- 可重複利用性：不限於特定的區塊。
- 可維護性：需求變更只需要修改部分程式碼。
- 可擴展性：。

## CSS 命名類型
- 純粹的心法
- 依賴工具而遵循（Boostrap、UI Library）

### CSS 心法
- OOCSS
- SMACSS
- BEM

### CSS 工具
- Boostrap

## CSS 命名陷阱
- 所見不為所得
- 唯一性的命名 => Boostrap color 中的 danger 為例
